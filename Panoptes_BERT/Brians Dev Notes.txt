Refreshing Bash Commands in the Script

Here's a quick overview of each command in the bash script you have:

    1. docker stop proj_dev_db: This command stops the running container named proj_dev_db. If the container is not running, it does nothing.

    2. docker rm proj_dev_db: This removes the stopped container so that you can reuse the name when creating a new container. It's important to remove the old container to avoid name conflicts and to clean up resources.

    3. docker build -t proj_panoptes_app .: Builds a Docker image named proj_panoptes_app from a Dockerfile in the current directory (. signifies the current directory).

    4. docker run --name proj_dev_db -p 3306:3306 -d proj_panoptes_app: Runs the newly built image as a container named proj_dev_db, forwarding the host's 3306 port to the container's 3306 port, and runs the container in detached mode (-d).

    5. sleep 30: Pauses the script for 30 seconds to allow MySQL in the Docker container enough time to initialize before attempting to execute SQL commands.

    6. docker exec proj_dev_db mysql -u root -pRum8&Tag2 -e "CREATE DATABASE newdatabase;" and the following command are used to execute SQL commands within the proj_dev_db container.
